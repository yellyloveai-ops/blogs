# AGENTS.md

Instructions for AI agents working in this repository.

## Repository Purpose

This is a blog repository for the yellyloveai-ops project. Articles cover AI agent runtime design, scheduling, self-spawning agents, and related topics around Claude Code and yelly-time.

## Content Guidelines

- All posts are Markdown files in the repo root
- Tone: concise, technical, practitioner-focused ("5 min read" style)
- No fluff — lead with the problem, then the solution
- Use `---` horizontal rules as section separators
- Each post should have a `# Title` and optional byline on line 2 (`*N min read · team*`)

## File Naming

Use lowercase with hyphens for new posts, e.g. `yelly-new-topic.md`. Existing files use underscores — don't rename them.

## What Agents Should NOT Do

- Do not modify `LICENSE`
- Do not delete or rename existing `.md` files
- Do not add build systems, package.json, or CI config unless explicitly asked
- Do not reformat existing posts

## Working on Posts

When drafting or editing a post:
1. Read at least two existing posts first to match the voice and structure
2. Keep posts between 500–1000 words
3. Use concrete examples over abstract descriptions
4. End with a clear takeaway or call to action
